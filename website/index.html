When markets close,
bStocks keep drifting.
BQuant measures the gap.

NYSE and Nasdaq halt every weekend. The bStock tokens that mirror them on BNB Chain do not. Over those closed hours, on-chain price and reference price diverge — and when markets reopen Monday, gravity pulls them back. BQuant tracks that drift in real time and emits a mean-reversion signal the moment the gap is wide enough to act on.

How it works

Drift formula & signal logic

BQuant's core idea: a bStock's on-chain price can drift from its NYSE/Nasdaq reference anchor while the underlying exchange is closed. When the drift exceeds a threshold, mean-reversion back to the reference anchor is the expected outcome at market open. Direction is contrarian — price drifted up means SHORT (it will fall back); price drifted down means LONG (it will rise back).

# Drift formula — drift_engine.py
drift_t = (price_t − price_t0) / price_t0

# Where:
# price_t0 = anchor price (first snapshot in data/weekend_snapshots.json)
# price_t = most recent CMC snapshot price
# drift_t = signed fraction; positive = price rose above anchor

The signal thresholds below are hard-coded in drift_engine.py and govern all five tokens uniformly. Confidence is an ordinal 0–100 band, not a statistical p-value.

Condition Signal Direction Confidence
|drift| > 1.5% LONG / SHORT Opposite to drift sign (reversion bet) 60 at threshold → 85 at 4.5%
|drift| < 0.3% CONVERGED Price returned to anchor — exit or skip Fixed 90
0.3% ≤ |drift| ≤ 1.5% FLAT Insufficient edge; no position Fixed 50

Mandatory signal output format

[Strategy: BQuant — bStock Weekend Drift]
[Token: <TICKER>]
[Anchor (t0): <price> @ <timestamp>]
[Current Price: <price> @ <timestamp>]
[Drift: x.xx%]
[Signal: LONG / SHORT / FLAT / CONVERGED]
[Confidence: xx%]
[Reasoning: <one sentence>]

Every signal emitted by the system — to terminal, dashboard, or on-chain call — must include this block verbatim. The logId emitted by BQuantSignalRegistry.logSignal() on BSC testnet links the on-chain event to this off-chain reasoning block.

SDK

Pluggable signal engine

drift_engine.py is a pure-function module with no I/O and no side effects — importable into any agent or orchestration layer. Two functions, JSON-serializable in/out, no exceptions on bad data.

calculate_drift()
calculate_drift(snapshots: list, token: str) → dict
Computes signed fractional drift between the anchor snapshot (index 0) and the most recent snapshot for a given token. Returns anchor price, current price, drift_pct, snapshot_count, and an ok/error field for missing tokens.
Returns: anchor_price · current_price · drift_pct · snapshot_count · ok · error
generate_signal()
generate_signal(drift_pct: float) → dict
Maps a drift fraction to LONG / SHORT / FLAT / CONVERGED with a confidence integer (0–100) and a one-sentence reasoning string. Pure function — stateless, no I/O.
Returns: signal · confidence · reasoning
import json, importlib.util

# Load engine (works from any calling context)
spec = importlib.util.spec_from_file_location("drift_engine", "skills/bquant-weekend-drift/scripts/drift_engine.py")
engine = importlib.util.module_from_spec(spec); spec.loader.exec_module(engine)

with open("data/weekend_snapshots.json") as f:
    snapshots = json.load(f)

drift = engine.calculate_drift(snapshots, "NVDAB")
# → {"token": "NVDAB", "anchor_price": 209.56, "drift_pct": -0.000353, "ok": True, ...}

signal = engine.generate_signal(drift["drift_pct"])
# → {"signal": "CONVERGED", "confidence": 90, "reasoning": "..."}

The engine is registered as skill bquant-weekend-drift in the CMC Skill Hub MCP context. Agents invoke it via the MCP tool interface; scripts can import it directly. Token list is the single source of truth in tokens.py.
